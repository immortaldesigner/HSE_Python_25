# Homework 3. URL Shorter

Стек технологий:
- FastAPI (Python 3.10)
- PostgreSQL (SQLAlchemy ORM)
- Redis (Асинхронный клиент для кэширования и Rate Limiting)
- Docker и Docker Compose
- JWT авторизация (passlib, python-jose)
- Pillow и QRcode (Генерация изображений)

## Основные функции
### Работа с пользователями
- POST /register — Регистрация нового аккаунта.
- POST /login — Авторизация и получение JWT токена.

### Управление ссылками
- POST /links/shorten — Создание короткой ссылки. Доступно для анонимных и авторизованных пользователей. Поддерживает custom_alias и установку даты истечения (expires_at).
- GET /{short_code} — Мгновенный редирект. Использует Redis для кэширования.
- GET /links/search — Поиск короткой ссылки по полному URL.
- PUT /links/{short_code} — Обновление оригинального URL (только для владельца).
- DELETE /links/{short_code} — Удаление ссылки (только для владельца).

### Аналитика и дополнительные возможности
- GET /links/{short_code}/stats — Статистика кликов, включая дату последнего доступа и список последних 5 переходов (IP адрес, браузер, время).
- GET /links/{short_code}/qr — Генерация и выдача QR-кода в формате PNG.

### Features
#### Асинхронная аналитика
Обновление счетчика кликов и запись детальной информации о переходе (IP, User-Agent) вынесены в BackgroundTasks. Это позволяет возвращать RedirectResponse немедленно, не дожидаясь завершения записи в базу данных.

#### Rate Limiting
Реализована защита от спама на эндпоинте создания ссылок. Ограничение составляет 10 запросов в минуту с одного IP-адреса. Состояние счетчиков хранится в Redis.

#### Двухуровневое кэширование
При переходе по короткой ссылке система сначала проверяет наличие URL в Redis. При промахе кэша данные берутся из PostgreSQL и сохраняются в Redis со сроком жизни 1 час. Кэш автоматически инвалидируется (удаляется) при обновлении или удалении ссылки.

#### Автоматическое управление данными
В приложении работает фоновый процесс, который каждые 60 секунд проверяет срок действия ссылок. Истекшие ссылки удаляются из основной таблицы и перемещаются в архивную таблицу expired_links.

### Структура базы данных
users — информация о пользователях и хеши паролей.
links — активные короткие ссылки и агрегированная статистика.
link_clicks — детальный лог каждого перехода (аналитика).
expired_links — архив удаленных или истекших ссылок.
